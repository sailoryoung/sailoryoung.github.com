Golang template使用

发表于 2016-04-23   |   分类于 Golang   |  

Golang 是谷歌公司开发的一种“系统级”编程语言。本篇文章只是记录一下它提供的官方 template 库的使用方法。

template 库介绍

golang官方提供了两个 template 库，一个是text/template，另一个是html/template。前者可以用来做任何文本的模板，包括用来做 代码生成器 之类的。后者则专用在做 html模板，提供了针对html,js和css的自动过滤输出机制，提高了安全性。其官方说明：

Package template (html/template) implements data-driven templates for generating HTML output safe against code injection. It provides the same interface as package text/template and should be used instead of text/template whenever the output is HTML.

因此本篇主要介绍text/template使用。

控制结构

典型控制结构有{{if pipeline}} T1 [{{else [if pipeline]}} T0] {{end}}、{{range pipeline}} T1 [{{else}} T0] {{end}}、{{with pipeline}} T1 [{{else}} T0 ] {{end}}三种。
其中：

  • {{if pipeline}} T1 [{{else [if pipeline]}} T0] {{end}}用于条件输出，判断标准为pipeline是否为空。不影响结构内{{.}}的值
  • {{with pipeline}} T1 [{{else}} T0 ] {{end}}可定义临时变量进行条件输出，判断标准为pipeline是否为空。结构内{{.}}的值会变为pipeline
  • {{range pipeline}} T1 [{{else}} T0] {{end}}用于循环输出，判断标准为当前输出键-值对的值是否为空。结构内{{.}}的值会变为当前输出键-值对的值。可以使用{{range $key,$value := pipeline}}定义临时变量输出键-值对。

模板复用

模板复用需要使用以下结构：

  • {{define "name"}} T {{end}} 定义一个模板。同名模板后定义的会覆盖之前的，此处仅为定义，不会渲染。
  • {{template "name" pipeline}} 使用已定义的模板，模板name中的{{.}}会被替换为pipeline
  • {{block "name" pipeline}} T {{end}} 定义并使用一个block，其中的{{.}}被替换为pipeline，block可被之后的{{define "name"}} T {{end}}重载。
